# 冷热钱包管理接口定义

## 版本说明

**当前版本：** v0.1.0 开发版

**说明：** 本文档定义冷热钱包管理相关的 REST API 接口，包括归档操作、多签审批、钱包配置管理等。

---

## 一、接口协议

### 1.1 通信协议

| 项目 | 说明 |
|------|------|
| 协议 | HTTP/1.1 |
| 数据格式 | JSON |
| 认证方式 | Bearer Token (JWT) |
| 字符编码 | UTF-8 |
| 基础路径 | `/api/v1` |

### 1.2 通用响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "trace_id": "abc123xyz"
}
```

### 1.3 错误码定义

| Code | 说明 |
|------|------|
| 0 | 成功 |
| 2001 | 钱包配置不存在 |
| 2002 | 归档金额不足 |
| 2003 | 归档阈值未达到 |
| 2004 | 审批不存在 |
| 2005 | 审批已过期 |
| 2006 | 签名数不足 |
| 2007 | 签名验证失败 |
| 2008 | 审批者无权限 |
| 2009 | 交易已广播 |
| 2010 | 钱包状态异常 |

## 二、钱包配置接口

### 2.1 创建钱包配置

**接口描述：** 创建新的钱包配置（热钱包或冷钱包）

**请求：**
```http
POST /api/v1/wallets/config
Content-Type: application/json
Authorization: Bearer <admin_token>

{
  "chain": "BTC",
  "wallet_type": "cold_wallet",
  "address": "bc1qxy2kgdygjrsqtzq2n0yrf2493p83kkfjhx0wlh",
  "config": {
    "sign_type": "multisig",
    "m": 3,
    "n": 5,
    "signers": [
      {"id": "signer_1", "pubkey": "02...", "name": "财务-张三"},
      {"id": "signer_2", "pubkey": "03...", "name": "财务-李四"}
    ]
  }
}
```

**响应：**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 1,
    "chain": "BTC",
    "wallet_type": "cold_wallet",
    "address": "bc1qxy2kgdygjrsqtzq2n0yrf2493p83kkfjhx0wlh",
    "status": "active",
    "created_at": 1680000000
  }
}
```

### 2.2 查询钱包配置

**接口描述：** 查询指定链的钱包配置

**请求：**
```http
GET /api/v1/wallets/config?chain=BTC&wallet_type=cold_wallet
Authorization: Bearer <token>
```

**响应：**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 1,
    "chain": "BTC",
    "wallet_type": "cold_wallet",
    "address": "bc1qxy2kgdygjrsqtzq2n0yrf2493p83kkfjhx0wlh",
    "config": {
      "sign_type": "multisig",
      "m": 3,
      "n": 5
    },
    "status": "active",
    "created_at": 1680000000
  }
}
```

### 2.3 更新钱包状态

**接口描述：** 更新钱包状态（如维护、停用）

**请求：**
```http

Content-Type: application/json

  "status": "maintenance",
```

## 三、归档操作接口

### 3.1 触发归档

**接口描述：** 手动触发热钱包到冷钱包的归档操作

**请求：**
```http
POST /api/v1/wallets/archive
Content-Type: application/json
Authorization: Bearer <admin_token>

{
  "chain": "BTC",
  "amount": "50000000",
  "reason": "manual"
}
```

**响应：**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "archive_id": "arc_20230601_001",
    "chain": "BTC",
    "from_address": "bc1q...hot",
    "to_address": "bc1q...cold",
    "amount": "50000000",
    "status": "pending",
    "created_at": 1680000000
  }
}
```

**业务规则：**
- 归档金额必须大于最小归档金额
- 归档后热钱包余额不低于保留金额
- 自动签名（热钱包）并广播交易

### 4.4 提交签名

**接口描述：** 审批者提交签名

**请求：**
```http
POST /api/v1/multisig/approvals/{approval_id}/sign
Content-Type: application/json
Authorization: Bearer <signer_token>

{
  "signature": "3044022076...",
  "signer_id": "signer_2"
}
```

**响应：**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "approval_id": "app_20230601_001",
    "current_signatures": 2,
    "required_signatures": 3,
    "status": "pending",
    "is_complete": false
  }
}
```

**业务规则：**
- 验证签名者身份
- 验证签名有效性
- 达到签名数后自动合并签名并广播交易
- 每个签名者只能签名一次

### 4.5 拒绝审批

**接口描述：** 审批者拒绝审批

**请求：**
```http
POST /api/v1/multisig/approvals/{approval_id}/reject
Content-Type: application/json
Authorization: Bearer <signer_token>

{
  "signer_id": "signer_1",
  "reason": "交易信息有误"
}
```

**响应：**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "approval_id": "app_20230601_001",
    "status": "rejected",
    "reject_reason": "交易信息有误"
  }
}
```

**业务规则：**
- 任一审批者拒绝，整个审批流程终止
- 关联的提现/归档操作状态更新为 rejected

## 八、接口调用流程示例

### 8.1 自动归档流程

```
1. 系统定时检测热钱包余额
   GET /api/v1/wallets/balances/hot?chain=BTC

2. 余额超过阈值，触发归档
   POST /api/v1/wallets/archive

3. 热钱包自动签名（HSM）
   （内部调用，无需接口）

4. 广播交易到链上
   （内部调用）

5. 监听确认，更新状态
   （后台任务）
```

### 8.2 大额提现多签流程

```
1. 用户发起大额提现
   POST /api/v1/withdrawals

2. 系统创建多签审批
   POST /api/v1/multisig/approvals

3. 通知审批者
   （邮件/钉钉/企微）

4. 审批者查询待审批
   GET /api/v1/multisig/approvals?status=pending

5. 审批者提交签名
   POST /api/v1/multisig/approvals/{approval_id}/sign

6. 达到签名数，自动广播
   （内部调用）

7. 推送完成事件
   （事件通知）
```
