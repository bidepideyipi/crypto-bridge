# 钱包服务接口定义

## 版本说明

**当前版本：** v0.1.0 开发版

**说明：** 本文档定义钱包服务层对外提供的 REST API 接口，供交易所业务层调用。开发版本通过测试用例模拟业务层请求，验证接口功能的正确性。

---

## 一、接口协议

### 1.1 通信协议

| 项目 | 说明 |
|------|------|
| 协议 | HTTP/1.1 |
| 数据格式 | JSON |
| 认证方式 | Bearer Token (JWT) |
| 字符编码 | UTF-8 |
| 基础路径 | `/api/v1` |

### 1.2 通用响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "trace_id": "abc123xyz"
}
```

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 响应码，0 表示成功 |
| message | string | 响应消息 |
| data | object/array | 响应数据 |
| trace_id | string | 请求追踪 ID |

### 1.3 错误码定义

| Code | 说明 |
|------|------|
| 0 | 成功 |
| 1001 | 参数错误 |
| 1002 | 用户不存在 |
| 1003 | 地址不存在 |
| 1004 | 余额不足 |
| 1005 | 交易失败 |
| 1006 | 链上确认失败 |
| 1007 | 冷钱包签名失败 |
| 1008 | 服务不可用 |
| 1009 | 链节点连接失败 |
| 1010 | 提现额度超限 |
| 1011 | 地址验证失败 |
| 1012 | 重复请求 |

## 二、地址管理接口

### 2.1 生成充值地址

**接口描述：** 为用户生成指定币种的充值地址

**请求：**
```http
POST /api/v1/addresses
Content-Type: application/json
Authorization: Bearer <token>

{
  "user_id": "user123",
  "chain": "BTC"
}
```

**响应：**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "address": "1A1zP1eP5QGefi2DMPTfTL5SLmv7DivfNa",
    "user_id": "user123",
    "chain": "BTC",
    "created_at": 1680000000
  }
}
```

**业务规则：**
- 同一用户同一链只能有一个活跃充值地址
- 地址生成后立即返回，无需等待链上确认

### 2.2 查询用户地址

**接口描述：** 查询用户已有的充值地址

**请求：**
```http
GET /api/v1/addresses?user_id=user123&chain=BTC
Authorization: Bearer <token>
```

**响应：**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "address": "1A1zP1eP5QGefi2DMPTfTL5SLmv7DivfNa",
    "user_id": "user123",
    "chain": "BTC",
    "active": true,
    "created_at": 1680000000
  }
}
```

### 2.3 查询地址详情

**接口描述：** 查询指定地址的详细信息

**请求：**
```http
GET /api/v1/addresses/1A1zP1eP5QGefi2DMPTfTL5SLmv7DivfNa
Authorization: Bearer <token>
```

**响应：**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "address": "1A1zP1eP5QGefi2DMPTfTL5SLmv7DivfNa",
    "chain": "BTC",
    "user_id": "user123",
    "balance": "1000000",
    "active": true,
    "created_at": 1680000000
  }
}
```

## 四、提现接口

### 4.1 创建提现申请

**接口描述：** 用户发起提现申请

**请求：**
```http
POST /api/v1/withdrawals
Content-Type: application/json
Authorization: Bearer <token>

{
  "user_id": "user123",
  "chain": "BTC",
  "to_address": "1BvBMSEYstWetqTFn5Au4m4GFg7xJaNVN2",
  "amount": "50000",
  "fee": "1000",
  "memo": "用户提现"
}
```

**响应：**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "withdraw_id": "wd_20230601_001",
    "status": "pending",
    "created_at": 1680000000
  }
}
```

**业务规则：**
- 提现金额必须大于最小提现额度
- 余额不足时返回错误码 1004
- 提现申请创建后进入待审批状态

**状态说明：**
- `pending`: 待审批
- `approved`: 已批准，等待签名
- `rejected`: 已拒绝
- `completed`: 已完成
- `failed`: 失败

### 4.2 查询提现状态

**接口描述：** 查询提现申请的当前状态

**请求：**
```http
GET /api/v1/withdrawals/wd_20230601_001
Authorization: Bearer <token>
```

**响应：**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "withdraw_id": "wd_20230601_001",
    "status": "completed",
    "tx_hash": "3b7ac3c6b11f9e8a7d2f1c4b5a6e8d9f0c1b2a3d4e5f6a7b8c9d0e1f2a3b4c5",
    "confirmations": 6,
    "created_at": 1680000000,
    "updated_at": 1680000600
  }
}
```

### 4.3 审批提现申请（内部接口）

**接口描述：** 审批人员或系统审批提现申请

**请求：**
```http
POST /api/v1/withdrawals/wd_20230601_001/approve
Content-Type: application/json
Authorization: Bearer <admin_token>

{
  "approved": true,
  "reason": ""
}
```

**响应：**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "withdraw_id": "wd_20230601_001",
    "status": "approved"
  }
}
```

### 4.4 查询用户提现记录

**接口描述：** 分页查询用户的提现记录

**请求：**
```http
GET /api/v1/withdrawals?user_id=user123&page=1&page_size=10
Authorization: Bearer <token>
```

**响应：**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "withdrawals": [
      {
        "withdraw_id": "wd_20230601_001",
        "chain": "BTC",
        "to_address": "1BvBMSEYstWetqTFn5Au4m4GFg7xJaNVN2",
        "amount": "50000",
        "status": "completed",
        "created_at": 1680000000
      }
    ],
    "total": 1,
    "page": 1,
    "page_size": 10
  }
}
```

## 六、回调接口（Wallet → Exchange）

### 6.1 充值事件通知

**接口描述：** 钱包服务检测到充值后，推送事件到交易所业务层

**事件格式（JSON）：**
```json
{
  "event_id": "evt_dep_20230601_001",
  "event_type": "deposit",
  "user_id": "user123",
  "chain": "BTC",
  "tx_hash": "a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6e7f8a9b0c1d2e3f4a5b6c7d8e9f0a1b2",
  "amount": "100000",
  "from_address": "1XPTgDRhN8RFnzniUGCdd1nG7MpoTraS2",
  "to_address": "1A1zP1eP5QGefi2DMPTfTL5SLmv7DivfNa",
  "confirmations": 6,
  "timestamp": 1680000600
}
```

**业务规则：**
- 消费端需基于 `event_id` 做幂等处理

### 6.2 提现事件通知

**接口描述：** 钱包服务完成提现后，推送事件到交易所业务层

**事件格式（JSON）：**
```json
{
  "event_id": "evt_wd_20230601_001",
  "event_type": "withdrawal",
  "withdraw_id": "wd_20230601_001",
  "user_id": "user123",
  "chain": "BTC",
  "tx_hash": "3b7ac3c6b11f9e8a7d2f1c4b5a6e8d9f0c1b2a3d4e5f6a7b8c9d0e1f2a3b4c5",
  "amount": "50000",
  "to_address": "1BvBMSEYstWetqTFn5Au4m4GFg7xJaNVN2",
  "status": "completed",
  "timestamp": 1680000600
}
```

**业务规则：**
- 消费端需基于 `event_id` 做幂等处理

## 九、错误响应示例

### 9.1 参数错误

**请求：**
```http
POST /api/v1/addresses
Content-Type: application/json

{
  "user_id": "user123"
}
```

**响应：**
```json
{
  "code": 1001,
  "message": "parameter error: chain is required",
  "data": null,
  "trace_id": "abc123xyz"
}
```

### 9.2 余额不足

**请求：**
```http
POST /api/v1/withdrawals
Content-Type: application/json

{
  "user_id": "user123",
  "chain": "BTC",
  "to_address": "1BvBMSEYstWetqTFn5Au4m4GFg7xJaNVN2",
  "amount": "999999999999"
}
```

**响应：**
```json
{
  "code": 1004,
  "message": "insufficient balance",
  "data": {
    "available": "100000",
    "required": "999999999999"
  },
  "trace_id": "abc123xyz"
}
```

## 十、HTTP 状态码说明

| HTTP Status | 说明 |
|-------------|------|
| 200 | 请求成功 |
| 400 | 请求参数错误 |
| 401 | 未授权（Token 无效或过期） |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |
| 503 | 服务不可用 |

**注意：** 业务错误使用 HTTP 200，通过响应体中的 `code` 字段区分。
