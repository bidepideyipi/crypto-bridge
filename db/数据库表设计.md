# 数据库表设计

## 版本说明

**当前版本：** v0.1.0 开发版

**存储方案：**
- **PostgreSQL**：主数据存储（地址映射、充值提现记录、余额账本）
- **Redis**：缓存层（链上余额缓存、限流计数、分布式锁）

---

## 一、PostgreSQL 表设计

### 1.1 用户地址表（user_addresses）

存储用户与钱包地址的映射关系。

```sql
CREATE TABLE user_addresses (
    id BIGSERIAL PRIMARY KEY,
    user_id VARCHAR(64) NOT NULL,
    chain VARCHAR(16) NOT NULL,
    address VARCHAR(128) NOT NULL,
    address_type VARCHAR(16) NOT NULL DEFAULT 'deposit', -- deposit, hot_wallet, cold_wallet
    status VARCHAR(16) NOT NULL DEFAULT 'active', -- active, inactive
    created_at BIGINT NOT NULL,
    updated_at BIGINT NOT NULL,
    UNIQUE (user_id, chain, address_type)
);

CREATE INDEX idx_user_addresses_user_id ON user_addresses(user_id);
CREATE INDEX idx_user_addresses_address ON user_addresses(address);
CREATE INDEX idx_user_addresses_chain ON user_addresses(chain);
```

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGSERIAL | 主键 |
| user_id | VARCHAR(64) | 用户 ID |
| chain | VARCHAR(16) | 链类型：BTC, ETH |
| address | VARCHAR(128) | 钱包地址 |
| address_type | VARCHAR(16) | 地址类型：deposit(充值), hot_wallet(热钱包), cold_wallet(冷钱包) |
| status | VARCHAR(16) | 状态：active(活跃), inactive(停用) |
| created_at | BIGINT | 创建时间（Unix 时间戳） |
| updated_at | BIGINT | 更新时间（Unix 时间戳） |

### 1.2 充值记录表（deposits）

存储链上充值交易记录。

```sql
CREATE TABLE deposits (
    id BIGSERIAL PRIMARY KEY,
    deposit_id VARCHAR(64) NOT NULL UNIQUE,
    user_id VARCHAR(64) NOT NULL,
    chain VARCHAR(16) NOT NULL,
    tx_hash VARCHAR(128) NOT NULL,
    from_address VARCHAR(128),
    to_address VARCHAR(128) NOT NULL,
    amount NUMERIC(36, 18) NOT NULL,
    status VARCHAR(16) NOT NULL DEFAULT 'pending', -- pending, confirmed, completed
    confirmations INTEGER NOT NULL DEFAULT 0,
    required_confirmations INTEGER NOT NULL,
    created_at BIGINT NOT NULL,
    updated_at BIGINT NOT NULL,
    completed_at BIGINT
);

CREATE INDEX idx_deposits_user_id ON deposits(user_id);
CREATE INDEX idx_deposits_tx_hash ON deposits(tx_hash);
CREATE INDEX idx_deposits_status ON deposits(status);
CREATE INDEX idx_deposits_chain ON deposits(chain);
CREATE INDEX idx_deposits_created_at ON deposits(created_at);
```

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGSERIAL | 主键 |
| deposit_id | VARCHAR(64) | 充值 ID（业务唯一标识） |
| user_id | VARCHAR(64) | 用户 ID |
| chain | VARCHAR(16) | 链类型：BTC, ETH |
| tx_hash | VARCHAR(128) | 链上交易哈希 |
| from_address | VARCHAR(128) | 来源地址 |
| to_address | VARCHAR(128) | 目标地址（充值地址） |
| amount | NUMERIC(36,18) | 充值金额（高精度） |
| status | VARCHAR(16) | 状态：pending(待确认), confirmed(已确认), completed(已完成) |
| confirmations | INTEGER | 当前确认数 |
| required_confirmations | INTEGER | 需要的确认数 |
| created_at | BIGINT | 创建时间（Unix 时间戳） |
| updated_at | BIGINT | 更新时间（Unix 时间戳） |
| completed_at | BIGINT | 完成时间（Unix 时间戳） |

### 1.3 提现记录表（withdrawals）

存储用户提现申请记录。

```sql
CREATE TABLE withdrawals (
    id BIGSERIAL PRIMARY KEY,
    withdraw_id VARCHAR(64) NOT NULL UNIQUE,
    user_id VARCHAR(64) NOT NULL,
    chain VARCHAR(16) NOT NULL,
    to_address VARCHAR(128) NOT NULL,
    amount NUMERIC(36, 18) NOT NULL,
    fee NUMERIC(36, 18) NOT NULL,
    status VARCHAR(16) NOT NULL DEFAULT 'pending', -- pending, approved, rejected, completed, failed
    tx_hash VARCHAR(128),
    confirmations INTEGER NOT NULL DEFAULT 0,
    required_confirmations INTEGER NOT NULL,
    memo TEXT,
    reject_reason TEXT,
    created_at BIGINT NOT NULL,
    updated_at BIGINT NOT NULL,
    approved_at BIGINT,
    completed_at BIGINT
);

CREATE INDEX idx_withdrawals_user_id ON withdrawals(user_id);
CREATE INDEX idx_withdrawals_status ON withdrawals(status);
CREATE INDEX idx_withdrawals_chain ON withdrawals(chain);
CREATE INDEX idx_withdrawals_tx_hash ON withdrawals(tx_hash);
CREATE INDEX idx_withdrawals_created_at ON withdrawals(created_at);
```

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGSERIAL | 主键 |
| withdraw_id | VARCHAR(64) | 提现 ID（业务唯一标识） |
| user_id | VARCHAR(64) | 用户 ID |
| chain | VARCHAR(16) | 链类型：BTC, ETH |
| to_address | VARCHAR(128) | 提现目标地址 |
| amount | NUMERIC(36,18) | 提现金额（高精度） |
| fee | NUMERIC(36,18) | 手续费（高精度） |
| status | VARCHAR(16) | 状态：pending(待审批), approved(已批准), rejected(已拒绝), completed(已完成), failed(失败) |
| tx_hash | VARCHAR(128) | 链上交易哈希 |
| confirmations | INTEGER | 当前确认数 |
| required_confirmations | INTEGER | 需要的确认数 |
| memo | TEXT | 用户备注 |
| reject_reason | TEXT | 拒绝原因 |
| created_at | BIGINT | 创建时间（Unix 时间戳） |
| updated_at | BIGINT | 更新时间（Unix 时间戳） |
| approved_at | BIGINT | 批准时间（Unix 时间戳） |
| completed_at | BIGINT | 完成时间（Unix 时间戳） |

### 1.5 余额变更流水表（balance_transactions）

记录所有余额变更的流水，用于对账和审计。

```sql
CREATE TABLE balance_transactions (
    id BIGSERIAL PRIMARY KEY,
    transaction_id VARCHAR(64) NOT NULL UNIQUE,
    user_id VARCHAR(64) NOT NULL,
    chain VARCHAR(16) NOT NULL,
    type VARCHAR(16) NOT NULL, -- deposit, withdraw, freeze, unfreeze, transfer
    amount NUMERIC(36, 18) NOT NULL,
    balance_before NUMERIC(36, 18) NOT NULL,
    balance_after NUMERIC(36, 18) NOT NULL,
    related_id VARCHAR(64), -- 关联的 deposit_id 或 withdraw_id
    created_at BIGINT NOT NULL
);

CREATE INDEX idx_balance_transactions_user_id ON balance_transactions(user_id);
CREATE INDEX idx_balance_transactions_chain ON balance_transactions(chain);
CREATE INDEX idx_balance_transactions_type ON balance_transactions(type);
CREATE INDEX idx_balance_transactions_related_id ON balance_transactions(related_id);
CREATE INDEX idx_balance_transactions_created_at ON balance_transactions(created_at);
```

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGSERIAL | 主键 |
| transaction_id | VARCHAR(64) | 流水 ID（业务唯一标识） |
| user_id | VARCHAR(64) | 用户 ID |
| chain | VARCHAR(16) | 链类型 |
| type | VARCHAR(16) | 变更类型：deposit(充值), withdraw(提现), freeze(冻结), unfreeze(解冻), transfer(转账) |
| amount | NUMERIC(36,18) | 变更金额（正数为增加，负数为减少） |
| balance_before | NUMERIC(36,18) | 变更前余额 |
| balance_after | NUMERIC(36,18) | 变更后余额 |
| related_id | VARCHAR(64) | 关联的充值或提现 ID |
| created_at | BIGINT | 创建时间（Unix 时间戳） |

## 二、Redis 缓存设计

### 2.1 链上地址余额缓存

缓存链上地址的实时余额，减少 RPC 调用。

```
Key: balance:{chain}:{address}
Value: {"balance": "1000000", "updated_at": 1680000000}
TTL: 60 秒
```

**用途：**
- 查询地址余额时优先读取缓存
- 60 秒 TTL 保证数据及时性

### 2.2 用户余额缓存

缓存用户账本余额。

```
Key: user_balance:{user_id}:{chain}
Value: {"balance": "1000000", "locked": "0", "updated_at": 1680000000}
TTL: 30 秒
```

**用途：**
- 快速查询用户余额
- 缓存失效时从数据库重新加载

### 2.3 充值事件去重

防止充值事件重复处理。

```
Key: deposit_event:{tx_hash}
Value: "1"
TTL: 24 小时
```

**用途：**
- 处理充值事件前检查是否已处理
- 基于链上交易哈希去重

### 2.4 提现限额控制

控制用户提现频率。

```
Key: withdraw_limit:{user_id}:{date}
Value: "3"  # 今日已提现次数
TTL: 当日 23:59:59
```

**用途：**
- 限制用户每日提现次数
- 防止频繁提现攻击

### 2.5 分布式锁

控制关键操作的并发。

```
Key: lock:withdraw:{user_id}
Value: "{request_id}"
TTL: 30 秒
```

**用途：**
- 防止用户并发提交提现
- 确保余额扣减的原子性

### 2.6 限流计数

API 调用限流。

```
Key: ratelimit:{user_id}:{endpoint}
Value: "10"  # 调用次数
TTL: 60 秒
```

**用途：**
- 限制用户 API 调用频率
- 防止恶意刷接口

### 2.7 链状态缓存

缓存区块链节点状态。

```
Key: chain_status:{chain}
Value: {"connected": true, "block_height": 792345, "lag": 2}
TTL: 10 秒
```

**用途：**
- 健康检查接口快速返回
- 监控链连接状态

## 三、数据访问策略

### 3.1 读写分离

```
写操作 → 主库
读操作 → 从库（延迟 < 1s）
```

**适用读操作：**
- 查询用户地址
- 查询充值/提现记录
- 查询余额流水

**必须走主库：**
- 用户余额扣减（需要最新值）
- 提现状态更新

### 3.2 缓存策略

| 数据 | 缓存 | 更新策略 |
|------|------|---------|
| 链上地址余额 | ✅ Redis | TTL 失效后 RPC 查询 |
| 用户余额 | ✅ Redis | 写入后更新缓存 |
| 充值事件去重 | ✅ Redis | 写入后设置 Key |
| 用户地址 | ❌ - | 直接查询数据库 |
| 提现记录 | ❌ - | 直接查询数据库 |

### 3.3 事务边界

**充值入账事务：**
```
BEGIN
1. 更新 deposits 状态
2. 插入 balance_transactions
3. 更新 user_balances
4. 清除用户余额缓存
COMMIT
```

**提现创建事务：**
```
BEGIN
1. 插入 withdrawals
2. 更新 user_balances（冻结余额）
3. 插入 balance_transactions
4. 清除用户余额缓存
COMMIT
```

## 四、数据归档策略

### 4.1 归档规则

| 数据类型 | 归档周期 | 保留策略 |
|---------|---------|---------|
| 充值记录 | 3 个月后 | 保留 3 年 |
| 提现记录 | 3 个月后 | 保留 3 年 |
| 余额流水 | 6 个月后 | 保留 5 年 |
| 链上交易追踪 | 1 个月后 | 保留 1 年 |

### 4.2 归档方式

- 将历史数据迁移到归档表或归档库
- 热表只保留近期数据
- 提供归档数据查询接口
